# 前后端代码职责文档

## 1. 文档目的

本文档用于说明当前 `seckill-system` 仓库中各目录、各服务、各分层代码分别负责什么功能，帮助前后端成员快速理解代码边界、协作关系和排查入口。

本文重点覆盖：

- 前端页面、组件、接口封装的职责
- 用户、商品、订单、库存四个后端服务的职责
- 网关、基础设施、数据库脚本与部署目录的职责
- 哪些目录属于业务代码，哪些目录属于生成物或本地环境

## 2. 仓库顶层目录职责

### `frontend/`

前端 Web 应用，基于 React + Vite。

负责：

- 商城首页展示
- 秒杀页面交互
- 登录注册界面
- 前端路由
- 调用后端商品、订单、用户接口

### `user_service/`

用户服务。

负责：

- 用户注册
- 用户登录
- 密码加密与校验
- JWT 令牌生成

### `product_service/`

商品服务。

负责：

- 商品列表查询
- 商品详情查询
- 商品搜索
- Redis 缓存预热
- 读写分离和缓存保护逻辑

### `order_service/`

订单服务。

负责：

- 秒杀下单入口
- 我的订单查询
- 单订单查询
- 订单支付入口
- 订单 Outbox 发布
- 消费库存结果和支付结果消息

### `inventory_service/`

库存服务。

负责：

- 秒杀库存预扣减
- Redis 限购与库存原子控制
- 库存预留、确认、取消
- 库存结果 Outbox 发布
- 消费订单创建事件

### `api_gateway/`

网关目录，当前代码层面基本还是示例入口。

负责：

- 预留统一 API Gateway 入口位置
- 当前版本真正的请求转发主要由 `nginx.conf` 和 Docker 编排完成

### `infra/`

基础设施目录。

负责：

- MySQL 初始化脚本
- MySQL 主从复制配置脚本
- ShardingSphere-Proxy 配置
- 容器化部署所需的基础镜像和配置

### `docs/`

项目文档目录。

负责：

- 记录功能设计
- 记录架构改造说明
- 记录前端视觉与交互规范
- 记录本次代码职责文档

### 根目录文件

#### `docker-compose.yml`

项目整体容器编排文件。

负责：

- 启动 MySQL、Redis、Kafka、Zookeeper、ShardingSphere
- 启动用户、商品、订单、库存服务
- 启动订单 worker、库存 worker
- 启动前端 Nginx

#### `nginx.conf`

统一反向代理与静态资源入口配置。

负责：

- 承载前端页面静态资源
- 将 `/api/users` 转发到用户服务
- 将 `/api/products` 转发到商品服务
- 将 `/api/orders` 转发到订单服务
- 将 `/api/inventory` 转发到库存服务
- 通过 Docker DNS 动态解析后端容器地址，避免容器重建后出现旧 IP `502`

## 10. 非业务目录说明

以下目录主要用于本地开发或构建产物，不属于核心业务代码：

- `frontend/node_modules/`
- `frontend/dist/`
- `user_service/venv/`
- `order_service/venv/`
- `inventory_service/venv/`
- 各服务下的 `.idea/`
- 各目录下的 `__pycache__/`

这些目录通常不用于阅读业务逻辑，排查功能时可以优先忽略。

## 11. 建议的阅读顺序

如果是新成员首次进入项目，建议按下面顺序阅读：

1. 先看根目录的 `docker-compose.yml` 和 `nginx.conf`，理解服务关系。
2. 再看 `frontend/src/App.jsx`、`frontend/src/pages/`，理解前端页面入口。
3. 再看 `user_service/main.py`，理解登录注册链路。
4. 再看 `product_service/app/main.py -> api -> application -> infrastructure`，理解商品查询链路。
5. 再看 `order_service/app/api/routes.py` 和 `order_service/app/application/services/order_service.py`，理解下单与支付链路。
6. 再看 `inventory_service/app/api/routes.py` 和 `inventory_service/app/services/inventory_service.py`，理解库存预扣减链路。
7. 最后看 `infra/` 和 `docs/`，理解数据库、分片和部署结构。
